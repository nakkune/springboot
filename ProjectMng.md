## 1. 제품 개요 및 목표

### 제품 비전

모든 팀이 스프레드시트 없이도 업무를 시각적으로 관리하고, 실시간으로 진행 상황을 공유할 수 있는 단일 협업 허브를 제공한다.

ProjectOS는 팀이 업무를 계획·추적·협업할 수 있는 Work OS 플랫폼입니다. Monday.com의 Board 중심 인터페이스와 유연한 워크플로우 자동화를 참조하여, 국내 기업 환경에 최적화된 프로젝트 관리 도구를 제공합니다.

### 핵심 문제 정의

업무가 이메일·메신저·엑셀에 분산되어 누락 발생, 진행률 파악 어려움, 온보딩 비용 과다.
→ 하나의 플랫폼에서 할당·추적·보고를 통합 처리.

### 범위 (In-Scope / MVP)

- Board 뷰, 칸반 뷰, 타임라인(간트) 뷰
- 워크스페이스 & 프로젝트 관리
- 자동화 기본 규칙
- 멤버 초대 & 권한 관리
- 알림 & 댓글
- 대시보드 위젯

## 2. 타겟 사용자 및 페르소나

### 페르소나 1 ? 프로젝트 매니저 (PM)

- 전체 일정·리소스·진행률 관리가 필요
- 간트 차트 & 대시보드 보고서 필수
- 자동화 규칙으로 반복 업무 제거 원함

### 페르소나 2 ? 개발자 / 실무 담당자

- 본인 할당 태스크를 빠르게 확인
- 칸반 보드로 상태 드래그 업데이트
- GitHub·Jira 등 기존 툴 연동 필요

### 페르소나 3 ? 경영진 / 팀 리더

- 포트폴리오 수준의 요약 현황 필요
- 리스크 & 지연 항목 빠른 파악
- 읽기 전용 대시보드 공유 기능 필요

## 3. 핵심 기능 명세

### 3.1 Board (스프레드시트 뷰)

- 행 = 아이템, 열 = 컬럼 (상태·담당자·날짜·숫자 등)
- 컬럼 타입 커스텀 (텍스트·드롭다운·체크박스·파일)
- 그룹 단위 행 묶음 / 인라인 편집
- 필터·정렬·검색 실시간 반영

### 3.2 칸반 뷰

- 상태 컬럼 기준 카드 자동 분류
- 드래그 & 드롭으로 상태 변경
- 카드 내 서브태스크·댓글·첨부파일
- WIP 제한 수 설정 지원

### 3.3 타임라인 (간트) 뷰

- 날짜 컬럼 기준 간트 바 자동 생성
- 의존성 화살표 (FS·SS·FF)
- 바 드래그로 일정 조정
- 크리티컬 패스 하이라이트

### 3.4 자동화 (Automation)

- Trigger → Condition → Action 규칙 빌더
- 상태 변경 시 담당자 자동 알림
- 기간 초과 시 우선순위 자동 상향
- Webhook · Slack · 이메일 액션

### 3.5 대시보드

- 번다운·파이·바·숫자 위젯
- 복수 보드 데이터 집계
- 공유 링크 (읽기 전용)
- 자동 새로고침 (5분 주기)

### 3.6 팀 & 권한 관리

- 워크스페이스 / 팀 / 프로젝트 계층 구조
- 역할: Owner · Admin · Member · Guest
- 보드별 접근 권한 개별 설정
- 이메일·링크 초대 / SSO 지원 (엔터프라이즈)

## 4. 사용자 스토리

| 페르소나 | 나는 ~하고 싶다 | 그래서 ~할 수 있다 | 인수 조건 |
|---------|---------------|-----------------|---------|
| PM | 보드에서 태스크에 담당자와 마감일을 지정 | 누가 언제까지 무엇을 해야 하는지 한눈에 파악 | 담당자 변경 즉시 알림 발송, 달력 뷰 반영 |
| 개발자 | 내 태스크만 필터링해서 칸반 뷰로 확인 | 오늘 할 일에만 집중 | 필터 저장·공유 가능, 칸반 카드 1초 내 로드 |
| PM | 상태가 '완료'로 바뀌면 자동으로 다음 담당자에게 알림 | 수동 보고 없이 워크플로우 자동 진행 | 자동화 규칙 UI에서 3분 내 설정 완료 가능 |
| 경영진 | 전체 프로젝트 진행률을 대시보드 하나로 확인 | 매주 보고 미팅 없이 현황 파악 | 공유 링크로 비회원도 읽기 전용 접근 가능 |
| 팀원 | 태스크에 파일·댓글·@멘션 남기기 | 맥락을 잃지 않고 비동기 협업 | 댓글 스레드, 파일 업로드(100MB), 멘션 알림 |

## 5. 기능 우선순위 (MoSCoW)

### Must Have (필수)

- Board 뷰 (아이템·컬럼 CRUD)
- 칸반 뷰 & 드래그앤드롭
- 멤버 초대 & 권한 관리
- 인앱 알림 & 댓글 & 멘션

### Should Have (권장)

- 타임라인 (간트) 뷰
- 자동화 규칙 빌더
- 대시보드 & 위젯

### Could Have (선택)

- Slack · Webhook 연동
- 외부 공유 링크 (읽기 전용)

### Won't Have (v1 제외)

- SSO / SAML 2.0
- AI 자동 태스크 생성

## 6. 비기능 요구사항 (NFR)

### 6.1 성능

- 페이지 로드 < 2초 (LCP 기준)
- 보드 1000행 스크롤 60fps 유지
- API 응답 P95 < 300ms

### 6.2 보안

- 데이터 암호화 (AES-256, TLS 1.3)
- OAuth 2.0 / SSO (SAML 2.0, 엔터프라이즈)
- 접근 로그 감사 (Audit Log)

### 6.3 가용성

- SLA 99.9% uptime
- 무중단 배포 (Rolling Deploy)
- 재해복구 RTO < 4h / RPO < 1h

### 6.4 확장성

- 워크스페이스당 최대 10만 아이템 지원
- 동시 접속자 10,000명 지원
- 수평 확장 가능한 마이크로서비스 구조

## 7. 마일스톤 로드맵

| Phase | 기간 | 내용 |
|-------|------|------|
| Phase 0 ? 기획 & 설계 | M1 | PRD 확정, 디자인 시스템 구축, 기술 스택 결정, DB 스키마 설계 |
| Phase 1 ? MVP | M2~M3 | Board 뷰 + 사용자 인증, 워크스페이스 생성, Board CRUD, 컬럼 타입(상태·담당자·날짜), 멤버 초대 |
| Phase 2 | M4~M5 | 칸반 드래그앤드롭, 간트 뷰, 댓글 & 멘션, 이메일/인앱 알림 시스템 |
| Phase 3 | M6~M7 | 자동화 규칙 빌더, 대시보드 위젯, 공유 링크, Slack·Webhook 연동 |
| Phase 4 ? 런치 | M8~M10 | SSO, 감사 로그, 고급 권한, 성능 최적화, 베타 → 정식 출시 |
